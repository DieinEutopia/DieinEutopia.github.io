Hexo中LaTeX无法被正确渲染的解决方案

发表于 2020-07-02 分类于 技术

配置插件

安装Hexo默认Mathjax渲染插件Marked

根目录下运行: 

npm install hexo-math --save

用Kramed替换Marked

Marked引擎是Hexo默认的渲染引擎，Kramed在此基础上修复了部分bug（但仍存在一些问题）

首先卸载Marked，根目录下运行： 

npm uninstall hexo-renderer-marked --save

然后安装Kramed，根目录下运行：

npm install hexo-renderer-kramed --save

放弃使用hexo-math

这个包已经被deprecated,不再被使用，故选择使用更新的包

根目录下分别运行：

npm uninstall hexo-math --save
npm install hexo-renderer-mathjax --save

解决语义冲突的问题

由于LaTeX与markdown语法有语义冲突，例如可用\* 表示斜体或加粗，可用 \_ 表示斜体，所以在此需要修改变量，将 _ 用于LaTeX，而保留使用 * 表示markdown中的斜体和加粗。

在根目录下进入 node_modules\kramed\lib\rules\inline.js

修改第11行escape变量的值（在原基础上取消了对，{，}的转义），如下：

escape: /^\\([\\`*{}\[\]()#$+\-.!_>])/, #原值 

escape: /^\\([`*\[\]()#$+\-.!_>])/, #复制本行内容替换之

修改第20行em变量的值，如下：

//  em: /^\b_((?:__|[\s\S])+?)_\b|^\*((?:\*\*|[\s\S])+?)\*(?!\*)/, #原值

em: /^\*((?:\*\*|[\s\S])+?)\*(?!\*)/, #复制本行内容替换之

更改配置文件

进入根目录的 _config.yml ，将mathjax默认的 False 修改为 True，并更换CDN的URL，如下：

# MathJax Support
mathjax:
enable: true
per_page: true
#cdn: //cdn.bootcss.com/mathjax/2.7.1/latest.js?config=TeX-AMS-MML_HTMLorMML
cdn: //cdnjs.cloudflare.com/ajax/libs/mathjax/2.7.1/MathJax.js?config=TeX-AMS-MML_HTMLorMML

文章编辑

在需要用LaTeX渲染的文章前（即Front-matter）打开mathjax开关，如下：

---
title:
date: 
tags:
mathjax: true #复制于此

测试

在编辑软件中输入以下内容：

$\left[\begin{array}{c}c^{\prime}(t) \\ k^{\prime}(t)\end{array}\right]=\left[\begin{array}{cc}0 & \frac{1}{\theta} f^{\prime \prime}\left(k^{\*}\right) c^{\*} \\ -1 & \beta\end{array}\right]\left[\begin{array}{c}c-c^{\*} \\ k-k^{\*}\end{array}\right]$

渲染结果：

$\left[\begin{array}{c}c^{\prime}(t) \\ k^{\prime}(t)\end{array}\right]=\left[\begin{array}{cc}0 & \frac{1}{\theta} f^{\prime \prime}\left(k^{*}\right) c^{*} \\ -1 & \beta\end{array}\right]\left[\begin{array}{c}c-c^{*} \\ k-k^{*}\end{array}\right]$

输出效果很nice

（ 注意：由于语义冲突，故必须在每个 * 前加上转义符 \ , 即 \* )
